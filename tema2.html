Коментарі <!-- -->

Коментарі розташовуються між тегами <!-- i -->. Коментарі використовують для пояснення коду. А також коментарі можна використовувати для того, щоб зробити певну частину коду не активною, для цього цей код треба взяти в тег коментаря. Коментарі можуть бути на одиин рядок, а можуть охоплювати декілька.

Кольори HTML

В HTML є можливість використовувати наступні кольори - RGB, HEX, HSL, RGBA, HSLA. RGB (red,green,blue)- rgb(0-255,0-255,0-255), (0,0,0) - чорний, (255,255,255) - білий, (у,у,у), (х,х,х) - відтінки сірого, інші значення це кольори. Повна палітра - 256*256*256 кольорів.
HEX, (#rrggbb) - rr- red, gg- green, bb- blue. (0-f,0-f,0-f,0-f,0-f,0-f), (#000000)- чорний колір, (#ffffff)- білий колір,(#xxxxxx),(#yyyyyy) - відтінки сірого, (#rrggbb) - кольори.
HSL (hue,saturation,lightness) (0-360, 0-100%, 0.0-1.0), hue-0-120=red,120-240=green,240-360=blue, saturation= 0- сірий, 100- колір, lightness= 0- чорний, 1- білий, 0,5 -колір.
RGBA,HSLA - кольори з альфа каналом, який відповідає за прозорість, вказується від 0.0 до 1.0- де 0 - повністю прозорий, а 1- повністю не прозорий.

HTML CSS

CSS може вказуватись в рядку елемента, в початковому тегу, після атрибута стайл. Може вказуватись в області хед, а також в окремому файлі, який підключається до сторінки через тег лінк.
Може використовуватись для того, щоб створити стилі для типових елементів на сторінці, може використовуватися для створення стилів класу, або для створення стилів елемента айді.

Посилання HTML

Посилання можуть бути абослютними (на зовнішні ресурси), чи відносними (на сторінки веб-сайту, або зображення веб-сайту, які знаходяться в каталозі веб-сайту).
Посиланням може бути зображення, текст. Те що взяте в тег посилання буде посиланням.

Зображення HTML

Зображення добавляється на сторінку за допомогою тегу img, атрибут src вказує на зображення в каталозі або поточній дерикторії, або на зовнішньому ресурсі.
Можна створити карту зображення і використовувати окремі області зображення як посилання. Для цього потрібно до тегу img добавити usemap="#mapname", а далі добавити тег <map name="mapname">
<area shape=rect; coord="x1,y1,x2,y2" href="page1.html">
<area shape=circle; coords="x,y,r" href="page2.html">
<area shape=poli; coords="x1,y1,...,xN,yN" href="psge3.html">
Ще можна добавити елемент picture, який буде підбирати з визначених зображень те, яке підходить по розміру вікна відображення.
Для цього потрібно додати тег picture, власти в нього тег source, а в ньому добавити атрибут media=(mix/max width:Npx;) а далі добавити srcset="img.jpg", так визначаючи мінімальну чи максимальну ширину екрану для наї ми встановлюємо зображення.
Для того, щоб зображення мало початкове значення, в не залежності від ширини вікне відображення ми добавляємо img з звичайними атрибутами (src, width, height, alt).

Favicon HTML

Для того, щоб додати іконку, яка буде відображатись в вкладці браузера перед описом, який вказується в head title, потрібно додати в head строку
<link rel="icon" type="image/x-icon" href="icon.png">

Таблиці HTML

Для того щоб додати таблицю ми використовуємо тег table, в таблиці може бути caption, colgroup, thead, tbody, tfoot, тобто голова, тіло, і підвал, також можуть бути tr,th,td, тобто рядки таблиці, заголовки таблиці, та комірки таблиці
Colspan i rowspan використовуються для того, щоб зробити комірку об'єднуючою декілька колонок чи рядків відповідно. Для цього colspan i rowspan вказуються в відкриваючому тегу елемента, який ми хочемо розширити на декілька стовпців чи рядків.
Також ми можемо використовувати тег стайл для того, щоб добавити межу в таблиці, між комірками, також ми можемо встановити колір для межі таблиці, і формат межі, пунктир (dashed), суцільна(solid), крапки (dotted), вдавлена(input), видавлена, (output), подвійна(double), для цього добавляємо style="border:1px solid blue;"
Ми можемо заокруглити кути комірок для цього використовується елемент border-radius:Npx; Для того щоб встановити ширину та висоту таблиці використовуються елементи height, width, які вказуються або в CSS, або в тегу table, або в th, td, для встановлення ширини та висоти заголовку чи комірки.

Списки

Списки можуть бути визначені, невизначені, а також описові.
Невизначені списки можуть позначатися кулею, кругом, або квадратом, або без позначення, для цього треба вказати bulet, circle, square, none в атрибуті list-style-type.
Невизначені списки часто використовуються для створення меню на веб-сайті, для цього прибирається позначення, а також вказується overflow:hidden; а для елементу li (list item) вказується float:left, а також для посилань в списку li a {display: block;}.
Визначені списки можуть бути марковані великими або маленькими буквами, цифрами, або римськими цифрами, великими або маленькими. Для цього вказується атрибут type з значенням A,a,1,I,i. Якщо потрібно почати маркування не з 1 а допустим з 100, то вказується атрибут start з значенням 100. Списки описання використовуються рідко, а для побудови списку потрібно використовувати dl - створення списку, dt - визначення терміну, dd - описання визначеного терміну.

Block & inline елементи HTML

Блокові елементи це елементи,які займають всю ширину рядка і завжди починаються з нового рядка. Такий елемент це div.
Лінійні елементи займають стільки місця, скільки потрібно, і продовжують рядок. span це лінійний елемент.
Ці елементи можна використовувати для форматування вмісту html сторінки

Класи HTML

Класи використовуються для надання стилів для любих елементів, які використовують клас.
Клас може використовувати декілька елементів сторінки, один елемент сторінки може використовувати декілька класів

Ідентифікатори HTML

Ідентифікатор може використовуватись для того, щоб задавати стиль для окремого елемента, який він ідентифікує. Або також може використовуватись для ідентифікації елемента для JavaScript

Фрейми HTML

Фрейми використовуються для відображення веб-сторінки всередині веб-сторінки.

HTML JavaScript

JavaScript може змінювати стилі, атрибути, може міняти картинки змінююючи значення атрибуту сорс. І ще багато чого вміє JavaScript, треба закінчити курс по JavaScript

Голова HTML

В head елементі розміщується інфморація про сторінку, обов'язковим є вказати кодування, яке використовує сторінка, а також додати тег meta viewport для адаптивності. Ще можна додати описання, ключові слова, авторство. А також в цей тег вкладають інфморацію про підключаємі бібліотеки, JavaScript, Bootstrap, а також тут додаються link для СSS, icon

Макет HTML

Для того, щоб створити макет існує декілька тегів, всі вони повинні форматуватися CSS, це header, nav, section, article, aside,footer. Ці теги використовуються для стилізації веб-сторінки.

Адаптивний веб-дизайн

Адаптивність досягається завдяки тегу в хед сторінки, meta viewport, який забезпечує відображення сторінки в розмірі вікна відображення присторою. Також можна зробити шрифт адаптивним, для цього його треба встановити в одиницях відображення vw, а не пікселях. Для адаптивності в елементах HTML вказують ширину і висоту в процентах.

Комп'ютерний код HTML

Комп'ютерний код це теги, які дозволяють визначитись між особливотями програмування і роботи програм, які можна використовувати для відображення коду на сторінці HTML. kbd - визначає введення з клавіатури code - визначає фрагмент комп'ютерного коду samp - визначає вибірку виводу з комп'ютерної програми var - визначає змінну в програмуванні або математичному виразі

Семантика HTML

Семантичні елементи, це ті елементи, які мають значення. Для html це header, nav, section, article, aside, footer

Юридичні сутності HTML, символи HTML, еможди HTML

Юридичні сутності, це наприклад можливість використання знаків, які потрібні підприємствам, наприклад знак власності на написаний текст, як інтелектуальна власність, ©. Сутності, це власне можливість відображати зарезервовані символи. Символів, які можна відобразити наразі близько 10 000, тому по простому варто користуватися довідником символьних сутностей.
Символи, це сутності, які вносяться до сторінки HTML через код, синтаксис коду наступний &entity_names, &#entity_numbers, наприклад & copy, <, >
Синатксис додавання емоджи такий самий.

Charset HTML

Для визначення кодування сторінки можна використовувати різні набори символів. Але в HTML 5 рекомендується використовуватиї UTF-8, який нараховує близько 10 000 символів.

URL
